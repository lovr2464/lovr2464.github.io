Git 基本操作

Git 是目前广泛使用的分布式版本控制系统，凭借其强大的功能和灵活性，受到了开发者的普遍好评。

我对 Git 的掌握程度并不高，在此我仅仅讲一些基本内容。

Git 的基本组成部分

Git 总体上对于用户分为以下几部分内容：

  • REPOSITORIES（仓库）：存储代码、历史记录和配置文件的地方，通常分为本地仓库和远程仓库。
  • BRANCHES（分支）：用于在同一个项目中并行开发的不同版本线，可以独立进行开发，最终合并到主线。
  • REMOTES（远程仓库）：远程托管的 Git 仓库，通常托管在 GitHub、GitLab 或 Bitbucket 上，方便团队协作。
  • STASHES（暂存区）：用于临时保存工作区的更改，可以稍后恢复或应用这些更改。
  • WORKTREES（工作区）：当前你正在操作的文件夹，其中包含了项目的所有文件。
  • COMMITS（提交）：记录在仓库历史中的每一个更改，每次提交都保存了代码库的一个新版本。
  • CONTRIBUTORS（贡献者）：记录项目中提交过代码的开发者，通常根据提交历史统计。
  • TAGS（标签）：用于标记特定提交的版本号或里程碑，通常用于发布软件版本。

在 Git 中，基本操作通常有克隆，推送，创建仓库、分支，暂存与提交

当然，Git 的命令并不仅限于这些。
